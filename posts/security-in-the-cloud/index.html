Security in the cloud

  1. AWS Compliance Programs: Amazon gets tested by many compliance programs. These can be global or not.

  2. AWS Artifact: Comprehensive list of cross-controlled documents relevant to compliance and security in the AWS cloud.

  3. AWS Shared Responsibility Model

  4. While AWS manages the security of the cloud, security in the cloud is the responsibility of the customer. Customers retain control of what security they choose to implement to protect their own content, platform etc.

  5. AWS is responsible for:

    1. Data Centers
    2. Regions
    3. AZ
    4. Hardware
    5. Edge locations
    6. Storage
    7. Data bases
    8. Networking
    9. Compute
    10. Hypervisor
    11. Patching RDS, SQL, OS’s

  6. Customer responsibilites:

    1. EC2 instances
    2. Client side data
    3. Server side data
    4. Things in S3 should be encrypted by default
    5. Networking (If you’re gonna be communicating over HTTPS or over a VPN etc)
    6. OS
    7. Firewalls
    8. IAM
    9. Patching EC2

  7. AWS WAF: Web application firewall that helps protect your web application from common web exploits.

  8. AWS Shield: Managed DDOS (Distributed Denial of Service) protection service that safeguards web applications running on AWS.

  9. WAF protects you against hackers, Shield protects you against DDOF attacks.

  10. DDOF: When someone sends you alot of malicious traffic, so much traffic that it causes our website to stop responding

  11. Shield is turned on automatically with all new AWS accounts

  12. Shield, 2 types:

    1. Standard: Comes with your AWS account
    2. Advance: More features, 3000$ per month
